// Edit a document. The redline, the PDF, the Word file,
// and the HTML update simultaneously. 
static func batchRedline(jobs: [RedlineJob]) async throws -> [RedlineResult] {
    try await withThrowingTaskGroup(of: RedlineResult.self) { group in
        for job in jobs {
            group.addTask {
                let start = CFAbsoluteTimeGetCurrent()
                let outputURL = try await redline(base: job.baseURL, modified: job.modifiedURL)
                let duration = CFAbsoluteTimeGetCurrent() - start
                return RedlineResult(label: job.label, outputURL: outputURL, duration: duration)
            }
        }
        var results: [RedlineResult] = []
        for try await result in group {
            results.append(result)
        }
        return results
    }
}

on complexity

Computer programmers as we know them today have existed for less than a century. They are not credentialed or licensed. They do not sit an exam.

And yet the things entrusted to computer programs are not somehow less important than the things entrusted to professionals who do. A Boeing 747 does not stay aloft because it is required by law to do so. It stays aloft because it is governed by computer programs that make it able to do so. How this is possible is worth understanding.

Over a remarkably short period, programmers have developed tools, practices, and protocols for building large-scale systems that interoperate with one another. iPhones are not less useful when three billion people have one. The tools do not degrade with scale.

Laws, by contrast, become harder to use when the number of people who must apply them increases. It is not obvious that this limitation reflects some fundamental constraint. It may simply reflect a gap in tooling.

If you print a document to Word and then print it to PDF, you have not done one thing twice. You have done two things once each.

Printing is printing. Redlining is redlining.

architecture
01
XPC Services
Document conversion and redline generation run in dedicated XPC services — separate processes with independent memory spaces, managed by launchd, created and destroyed on demand.
02
Parallel Redlining
The redline engine implements a word-level Myers diff. Multiple redlines execute in parallel — each with its own NSXPCConnection, its own handler, its own coordinated file access via NSFileCoordinator.
03
Zero-Copy IPC
No document content crosses the XPC boundary. The app passes file paths; the services read, process, and write directly to the filesystem.

Privacy is not a trade-off. It is a design constraint.

Separate processes. Separate memory. No content on the wire.

conventional vs. redliner
Print to Word, then print to PDF — two jobs
One edit propagates to HTML, PDF, and DOCX simultaneously via XPC
Run a redline manually after each revision
Redlines exist as a matter of process — they are produced, not requested
Choose between privacy and performance
XPC isolation — separate process memory, no content on the wire, no trade-off
Wait for one conversion to finish before starting the next
Parallel execution — ten redlines at once, each in its own process
